APIキー・トークン伏字（Redactor）ガイド

ログ共有前に「鍵っぽい文字列」を見つけて伏字化（保存なし）。

広告枠（準備中）

このツールでできること

  • ログ/設定/JSON/curlなどのテキストから、機密っぽい文字列（APIキー・トークン・Authorization等）を見つけやすくする
  • 共有用に伏字化して、貼り付けミスの確率を下げる（最終確認は人間）
  • ブラウザ内で処理（保存なし）
ツールを開く English guide

こんなときに使う

  • GitHub Issue / Discord / フォーラムにログを貼る前に、秘密情報が混ざってないか確認したい
  • Authorization: Bearer ... / api_key=... を消し忘れそう
  • .env や設定ファイルの一部を、なるべく安全に共有したい

関連キーワード：api key redaction / token masking / secret sanitize / bearer token

使い方

  1. ツール画面を開く
  2. 共有したいテキストを貼る
  3. 伏字後のテキストをコピーして共有する

コツ：共有前に、伏字後テキストを上からざっと目視チェック。

注意（できないこと・限界）

  • 伏字漏れゼロは保証できません。最終確認は必ず目視で行ってください。
  • 本当に危ない情報を外部に貼ってしまった場合は、キー/トークンの再発行（ローテーション）を検討してください。
  • このツールは「検出補助」です。100%の安全を担保するものではありません。
