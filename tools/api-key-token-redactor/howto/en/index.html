API Key & Token Redactor Guide

Quick usage, supported patterns, findings, limitations, and privacy notes.

30-second guide

  1. Paste logs, .env files, JSON, curl commands, or config snippets into the tool.
  2. Click “Redact” and review the findings list and safety summary.
  3. Copy or download only the redacted output before sharing.

Tip: search for sk-, github_pat_, xoxb-, AIza, AKIA, Authorization, or Cookie before posting.

Supported patterns

  • OpenAI: sk-, sk-proj-
  • GitHub: github_pat_, ghp_, gho_, ghu_, ghs_, ghr_
  • Slack: xoxb-, xoxp-, xapp-
  • Google API keys: AIza... / SendGrid: SG.... / GitLab: glpat-
  • AWS: AKIA..., ASIA..., aws_secret_access_key=
  • Bearer / Authorization headers, JWTs (eyJ...), and Cookie headers
  • PEM private keys (BEGIN/END PRIVATE KEY)
  • .env / JSON values labeled api_key, token, secret, password, client_secret, access_token, and similar names
  • URL query values such as ?token=, ?api_key=, and ?access_token=

How to read findings

  • High: private keys, Bearer tokens, JWTs, and major API keys.
  • Medium: labeled secrets, URL tokens, cookies, and values that need manual review.
  • Line: the approximate line number in the original text. Long single-line logs still need manual review.

What it doesn’t do

  • No guarantee of catching every secret.
  • Doesn’t scan images or screenshots.
  • Doesn’t validate or revoke tokens.
  • Doesn’t fully remove personal identifiers such as emails, IPs, user IDs, or full URLs.

Privacy

Runs locally in your browser. Nothing is uploaded.
