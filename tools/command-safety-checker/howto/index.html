Command Safety Checker：ガイド

AIコマンドを実行前に貼るだけ。危険パターンを検出してリスクを表示（保存なし）。

広告枠（準備中）

このツールでできること

  • bash / zsh / PowerShell のコマンドを貼り付けて、危険な実行パターンを検出
  • 破壊（削除/上書き）、漏洩（送信/出力）、権限過多（sudo/管理者）などをチェック
  • リスクを Low / Medium / High で可視化（最終判断は人間）
  • ブラウザ内で完結（保存なし）
ツールを開く English guide

こんなときに使う

  • ChatGPTやCopilotが出したコマンドを、そのまま叩くのが怖い
  • rm -rf や sudo などが混ざってないか事前に見たい
  • 社内PCや本番環境で実行する前に、最低限の危険チェックをしたい

関連キーワード：command safety, dangerous shell command, rm -rf, sudo, powershell safety, ai command check

使い方

  1. ツール画面を開く
  2. 実行予定のコマンド（複数行OK）を貼る
  3. 表示されたリスク（Low/Medium/High）と指摘内容を読む
  4. 不安が残る場合は、実行前にパス/対象/権限/環境を見直す

コツ：まずはテスト環境（ダミーディレクトリ）で動作確認。

注意（できないこと・限界）

  • 安全を保証するものではありません（最終判断は必ず人間）。
  • 一見安全でも、変数展開・パイプ・リダイレクト・権限・実行場所によって結果が変わります。
  • ネットワーク送信や環境依存の挙動は、静的チェックでは見落とす可能性があります。
  • 本番環境での実行は、バックアップや権限の確認を前提にしてください。
