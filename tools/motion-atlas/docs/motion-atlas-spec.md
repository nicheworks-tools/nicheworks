# Motion Atlas

**製品区分:** 有料化可能な実用Webツール
**運営名義:** Niche Works
**製品カテゴリ:** アニメーション一覧 / モーション辞典 / 選定支援ツール / AI依頼補助ツール
**対応言語:** 日本語優先
**対象:** Web制作初心者、AIでサイト制作する人、LP制作者、フロントエンド初中級者、UI改善担当者、個人開発者

---

## 1. 製品の定義

Niche Works Motion Atlas は、WebサイトやUIで使うアニメーションを、
**見本・用途・非推奨用途・強さ・軽さ・アクセシビリティ配慮・AI依頼文・実装メモ**
まで一体化して整理した実用アニメーション一覧である。

単なる「動きの見本帳」ではない。
単なる「CSSアニメーション集」でもない。
単なる「コードコピーツール」でもない。

この製品の価値は、次の8つを**1項目の中で完結**させることにある。

1. どういう動きか、その場で見て分かる
2. 何に向いている動きか分かる
3. どんな場面では使うべきでないか分かる
4. 似た動きとの違いが分かる
5. 派手さ・軽さ・スマホ相性が分かる
6. AIにどう依頼すればよいか分かる
7. 実装時に何を意識すべきか分かる
8. 選定の失敗を減らせる

## 2. 目標

### 2-1. ユーザー目標

ユーザーはこの製品を使って、次の問題を解決できること。

* 「少し動きを付けたい」が曖昧なまま止まる状態を抜けられる
* 見出し、CTA、カード、ローディングなど、**目的に合う動き**を選べる
* 派手すぎる / 地味すぎるを避けられる
* AIにアニメーションを正確に依頼できる
* モバイルで重い動きや不快な動きを避けられる
* reduce motion や a11y 観点を踏まえて選べる
* 実装判断をする前に候補比較ができる
* コードを書かなくても、必要な挙動の名前と方向性を把握できる

### 2-2. 事業目標

Niche Works にとっては、以下を狙う。

* SNS映えしやすい動く一覧ツールの構築
* SEOで「〇〇アニメーション」「テキストアニメーション」「ホバー演出」需要を拾う
* 無料で流入を取り、有料で比較・保存・コード展開・実装深掘りを売る
* UI Atlas と Vibe Lexicon から回遊させる
* Niche Works の「実用辞典シリーズ」の一本として資産化する

---

## 3. 非目標

この製品でやらないことを明確にする。

* After Effects テンプレ配布サイトにはしない
* 動画編集用のモーション素材集にはしない
* 3D演出や重厚なWebGL作品集にはしない
* 純粋なコードスニペット集が主役ではない
* デザイナー向けのショーケース特化サイトにはしない
* ただ派手な動きを並べるギャラリーにはしない
* ゲーム演出辞典にはしない

この製品は、あくまで
**Web制作で使うアニメーションの選定と依頼を助ける一覧**
に集中する。

## 9. 各アニメーションページの構成

### 9-1. ヘッダー領域

* アニメーション名
* 英語名
* 別名
* カテゴリ
* 強さタグ
* 軽さタグ
* モバイル相性タグ
* a11y配慮タグ

### 9-2. 一言説明

20〜40文字で要点を示す。

### 9-3. メインデモ

静止画ではなく、必ず動く。
ただし、演出を盛るのではなく、**どういう動きか瞬時に理解できること**を優先する。

### 9-4. 向いている場面

短文で3〜5行。
例:

* ヒーロー見出しに控えめな存在感を付けたい時
* LPで最初の印象を少し高めたい時
* 情報が少ないセクションの導入時

### 9-5. 向いていない場面

必須。
例:

* 長文の本文
* 同一画面内の大量要素
* 頻繁に再描画される領域
* 強いa11y配慮が必要なUI

### 9-6. 似た動きとの違い

比較表を置く。

例:

| 比較対象        | 主な違い    | 選び分け        |
| ----------- | ------- | ----------- |
| Fade up     | 距離が短く自然 | 無難に使いたい時    |
| Slide up    | 移動感が強い  | 動きを見せたい時    |
| Mask reveal | 演出感が強い  | ヒーローなど見せ場向き |

### 9-7. AIにそのまま使える依頼文

用途別に複数用意する。

* 基本版
* LP向け版
* UIコンポーネント向け版
* モバイル配慮版
* reduce motion 版

### 9-8. 実装メモ

* どのプロパティ中心か
* transform主体か
* opacity主体か
* layout shiftを起こしやすいか
* ループ向きか
* hover向きか
* scroll連動で重くなりやすいか
* durationの推奨幅

### 9-9. よくある誤用

例:

* 本文全部をタイプライターで出す
* CTAに常時強いパルスを付ける
* 長い距離のスライドをスマホで多用する
* 重要な通知を派手な弾みで見せる

### 9-10. 関連アニメーション

* 近い動き
* より控えめな代替
* より派手な代替
* a11y向け代替

## 18. デザイン仕様

この製品自体の見た目が弱いと説得力が落ちる。
ただし、派手すぎると辞典として疲れる。

### 18-1. デザイン方針

* デモが主役
* しかし比較と検索のしやすさを優先
* 演出サイトではなくツールサイト
* 情報が見やすい
* 長時間眺めても疲れにくい
* 強さと軽さが一目で分かる

### 18-2. トーン

Niche Worksらしく、

* 実用
* クリーン
* 少し未来感はあるが過剰ではない
* ツール感が強い
* モーション見本が埋もれない

### 18-3. 視覚上の重要要素

## 32. この製品で金を取れる理由

この製品が売るものは、アニメーション知識そのものではない。

売るのは、

* どの動きを選ぶか迷う時間
* AIに頼み直す時間
* 重い / うるさい / 不快な動きを試して失敗する時間
* 似た動きの違いを確認する時間
* 再利用したい設定を毎回作り直す時間

の削減である。

つまり、
**Web制作における“動き選びの摩擦”を減らすための有料道具**
として成立させる。
