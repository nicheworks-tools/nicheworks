# design-spec.md  
## 世界の建設・現場用語辞典 / Construction Tools & Slang Atlas  
### デザイン仕様（Notionクローンベース）

---

## 1. デザインコンセプト

- ベースは **Notion風のクリーンなドキュメントUI** を丸パクリしてよい。
- 「ツールっぽいWebアプリ」ではなく、  
  **“1枚のノートページの中に辞典UIが埋まっている”** 雰囲気にする。
- 余白広め・線は極力薄く・罫線少なめ・影も弱く。
- 世界観は **「Notionに貼られた建設用語Atlas」**。

追加で意識するのは：

- **タイトルと説明は完全にNotion風の「ページヘッダー」扱い。**
- 辞書カードや検索ブロックは「Notionのブロックが並んでいる」ように見せる。
- UIパーツはできるだけ **フラット・シンプル** に統一。

---

## 2. 全体レイアウト

### 2.1 ページ幅・センタリング

- 背景色：`#f7f7f5` 〜 `#fafaf8` のごく薄いオフホワイト（Notion風）
- メインコンテンツは **中央寄せの1カラム**。
- 最大幅：`max-width: 960px;`  
  （PCでは左右にしっかり余白が出るように）
- 画面左右は何も置かない（Notionのサイドバー非表示モード風）。

```text
[ 背景                     ]
[      ┌──────────────────┐]
[      │   ページヘッダー  │]
[      │   検索ブロック    │]
[      │   カテゴリブロック│]
[      │   結果リスト      │]
[      └──────────────────┘]
````

### 2.2 ブレークポイント

* 最小幅：360px（スマホ）
* 768px以上：PC/タブレットレイアウトとして扱う
* PC/スマホとも「1カラム」基本で、
  カテゴリボタンが2列→1列に崩れる程度の変化でOK。

---

## 3. タイポグラフィ

### 3.1 フォント

* ベースフォント（共通）：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI",
             system-ui, sans-serif;
```

### 3.2 見出し

* ページタイトル（NotionのH1相当）：

  * フォントサイズ：`32px` 前後
  * 太さ：`600`
  * 行間多め、上下に余白（上 32px / 下 8px）

* サブタイトル（説明文）：

  * `14–15px`
  * 色：`#6b7280`（mutedグレー）

* セクション見出し（「検索」「カテゴリ」「用語一覧」など）：

  * `18px`
  * 太さ：`600`
  * 下に 8〜12px の余白

---

## 4. カラーパレット（Notion寄せ）

```css
:root {
  --page-bg:    #fafaf8;   /* 全体背景（Notionの紙っぽい色） */
  --content-bg: #ffffff;   /* コンテンツ領域・カード背景 */
  --text-main:  #111827;   /* メインテキスト */
  --text-muted: #6b7280;   /* 補助テキスト */
  --border-subtle: #e5e7eb; /* うっすらしたボーダー */
  --tag-bg:     #f3f4f6;   /* タグ・チップ背景 */
  --tag-border: #e5e7eb;
  --accent:     #2563eb;   /* リンクや強調（NicheWorks共通ブルー） */
  --accent-soft:#eff6ff;   /* アクセントの薄い背景 */
}
```

* 建設っぽい黄色は **「危険ツールタグ」など局所だけ」に使う想定** とし、
  基本は Notion寄せのグレー＋ブルーでまとめる。

---

## 5. ページヘッダー（Notion風）

### 5.1 構成

* 上部に小さめの絵文字/アイコン（任意）

  * 例：🔧 や 🏗 など1文字
* その下に日本語タイトル（H1）
* さらに下に短い説明文（2〜3行）
* 説明文の下に、言語スイッチなどのコントロール群

```text
[🔧]
[世界の建設・現場用語辞典]
[道具・工具・作業名・特殊用語アトラス]
[ 説明テキスト…（1〜3行） ]
[ JP | EN   （トグル）      ]
```

### 5.2 言語切替

* Notionの「Property」的に、シンプルな pill ボタン2つ：

```text
[ JP ]  [ EN ]
```

* アクティブ側は背景を `var(--accent-soft)`、文字色を `var(--accent)`。

---

## 6. 検索ブロック（Notionのブロックっぽく）

### 6.1 テキスト検索バー

* 背景：`#ffffff`
* 枠：`1px solid var(--border-subtle)`
* 角丸：`8px`
* 中に虫眼鏡アイコン + プレースホルダー

```text
[ 🔍  工具名・作業名・スラングで検索 … ]
```

* フォーカス時：

  * 枠色を `var(--accent)` に
  * 影は極薄 `0 0 0 1px` レベルにしてNotionっぽく。

### 6.2 補助説明テキスト

* 検索バーの下に、ごく薄い小文字説明：

```text
例）「インパクト」「丸ノコ」「drill」「demo」など
```

---

## 7. カテゴリブロック（Notionのボタン集合）

### 7.1 表示形式

* Notionでタグプロパティを並べたような **pill型チップ** の集合。
* 横スクロールではなく、2〜3列の自動折返しグリッド。

```text
[ 切断 ] [ 締付 ] [ 穴あけ ] [ 計測 ]
[ 電動 ] [ 手動 ] [ 安全 ]   [ 解体 ]
```

### 7.2 スタイル

```css
.category-chip {
  display: inline-flex;
  align-items: center;
  padding: 4px 10px;
  border-radius: 9999px;
  background: var(--tag-bg);
  border: 1px solid var(--tag-border);
  font-size: 13px;
  color: var(--text-muted);
}
.category-chip--active {
  background: var(--accent-soft);
  border-color: var(--accent);
  color: var(--accent);
}
```

* アイコンはなくてもよいが、将来的に 🪚, 🔩 など絵文字を置けるよう余白確保。

---

## 8. 影絵アイコン（Silhouette Search）ブロック

### 8.1 配置

* カテゴリの下に「影絵で探す」的なサブ見出しを置き、その下に大きめの影絵ボタンを並べる。
* 1行あたり4つ（PC）、2つ（スマホ）。

```text
影絵で探す
[  🪚 丸ノコ系 ] [ 🔨 ハンマー系 ] [ 🪛 ドライバー系 ] [ ⚙️ その他 ]
```

### 8.2 スタイル

* ボタン本体は Notionのトグルブロックっぽくシンプルなカード。

```css
.silhouette-btn {
  background: var(--content-bg);
  border-radius: 12px;
  border: 1px solid var(--border-subtle);
  padding: 10px 12px;
  display: flex;
  flex-direction: column;
  align-items: flex-start;
  gap: 4px;
  cursor: pointer;
}
.silhouette-btn:hover {
  background: #f9fafb;
}
```

* 上にSVGアイコン or 絵文字、下に簡単なラベル（「丸ノコ系」など）。

---

## 9. 用語カード（Result Card）

### 9.1 レイアウト

Notionの「Page List」＋「Property」風：

```text
┌──────────────────────────────┐
│ 🪚 丸ノコ / Circular Saw       │ ← タイトル行（JP / EN）
│ 木材を高速回転の刃で…        │ ← 説明（アクティブ言語のみ表示）
│ [切断] [木材] [電動]           │ ← タグ群
└──────────────────────────────┘
```

* カード背景：`#ffffff`
* カード枠：`1px solid var(--border-subtle)`
* 角丸：`12px`
* 影：基本なし（Notion感を保つ）

### 9.2 スペーシング

* 内側：`padding: 10px 12px;`
* カード間：`margin-bottom: 8px;`
* タイトルと説明の間：`4px`
* 説明とタグの間：`6px`

### 9.3 タグ

* カテゴリタグ：カテゴリチップと同じ pillデザイン
* 危険度などを後付けする場合は、色タグ追加（例：危険→薄い赤背景）だが、MVPでは未使用でもよい。

---

## 10. スマホ対応（360px）

* すべて1カラム。
* 検索バーは画面幅ほぼいっぱいに。
* カテゴリチップは横スクロールではなく、折り返し。
* 影絵ボタンは2列グリッド→`minmax(0, 1fr)`で2分割。
* 上下余白は少し詰めるが、窮屈にしない。

---

## 11. インタラクション（最小限）

* ホバーは色差だけで、アニメーションは0.1〜0.15秒のフェード程度。
* クリック感はボーダー色変更などで表現。
* 「Notionっぽい＝動きすぎない・静かで落ち着いたUI」を維持。

---

## 12. NicheWorks 共通仕様との接続

* 最上部 or 下部に NicheWorks 共通ヘッダー/フッターを入れる場合も、
  **中身のコンテンツ領域は「Notionページ1枚」という前提で中央寄せする。**
* 余計なカード枠・パネルを増やさない。

---

## 13. MVPで必ず入れる要素

1. Notion風ページヘッダー（アイコン＋タイトル＋説明）
2. テキスト検索バー（Notion風入力欄）
3. カテゴリチップ（タグ風）
4. 用語カード（リスト）
5. スマホ360pxで崩れないレイアウト

影絵ブロックは**MVPに入れてもよいが、
最悪「スペースだけ作っておいて後日実装」でもOKな構成**にする。
