# CSV Tidy – QA Checklist (CSVTDY-24)

目的: 「整形→取り込み」が実務で通ることの最終確認。

## 事前
- [ ] ネットワーク送信無し（DevTools Networkで外部リクエストが出ない）
- [ ] 360 / 768 / 1024 / 1440 の4幅でUIが破綻しない

## テストケース

### 1) 会計CSV（Shift_JIS想定）
- [ ] Shift_JISのCSVを読み込み → 文字化け無し（またはエラーが次アクション付き）
- [ ] UTF-8出力でDL → Excel/Sheetsで開ける

### 2) EC CSV（列多め・不要列多い）
- [ ] 20〜100列のCSVで「除外/並べ替え/ヘッダ名変更」
- [ ] DLして取り込み先で列が想定通り

### 3) 名簿CSV（空白/表記ゆれ）
- [ ] 空白正規化 + トリムの効果確認
- [ ] 全半変換「選択列のみ」でID列だけ変換、住所列は変換しない

## 出力互換
- [ ] delimiter: 入力のまま / カンマ / TAB / セミコロン の4パターン
- [ ] quote: 必要時のみ / 常にクォート の2パターン
- [ ] 改行: LF（既定）/ CRLF（必要なら追加確認）

## 既知の注意
- 大容量CSVはブラウザ性能に依存（軽量モード/行数制限が必要なら別タスク化）
