広告枠

1. このツールでできること

地図ビューの要件を整理し、storm / compare / card 向けの確認メモを作るための下書きを生成します。

2. 使い方の流れ（5ステップ）

  1. モード（storm / compare / card）を選択します。
  2. 対象範囲（BBoxまたは緯度経度）と期間を入力します。
  3. レイヤーやメモなど、レビュー時に必要な情報を追記します。
  4. 「プレビューを作成」を押して結果を確認します。
  5. テキスト・PNG・CSVをコピー/ダウンロードして共有します。

3. 入力項目

  • BBox：対象範囲を「経度,緯度,経度,緯度」で指定（storm / compare）。
  • 緯度 / 経度：ポイント指定（card モード）。
  • 開始日 / 終了日：確認対象の期間（compare は A/B の2期間）。
  • プリセット：low / mid / detail のいずれか。
  • フレーム数（storm）：stormモードのみ必須。
  • 対象エリア：地図の対象範囲や地点。
  • 確認したいレイヤー：交通・地形・避難所などの情報。
  • 補足メモ：レビュー時に伝えたい注意点。

4. プレビュー結果の見方

  • storm：GSMaP実データのメタデータ状態パネルと、別枠の合成フレームスライダー/累積表示が出ます。
  • compare：Period A / Period B それぞれのGSMaP metadata到達性パネルと、別枠のA/B/Diff合成プレビューカード、CSV出力が出ます。
  • card：選択地点周辺のGSMaP metadata到達性パネルと、別枠の地点カード/ミニチャート/サマリー（合成プレビュー）が出ます。

storm / compare / card のメタデータ状態は実GSMaP metadata到達性を示しますが、表示される地図・地点カード・ミニチャート・サマリーは合成プレビューです。実データとして確認しているのはmetadata到達性だけです。ラスター降水量値はまだサンプリングしていません。

4a. データの状態

  • stormモードは /api/earth-map-suite/precipitation を1回呼び出し、GSMaP の実メタデータ到達性を表示します。
  • compareモードは同じエンドポイントをPeriod A / Period Bで別々に呼び出し、metadata-onlyの到達性だけを表示します。
  • cardモードは選択緯度経度の周辺bboxを作って同じエンドポイントを呼び出し、metadata-onlyの到達性だけを表示します。
  • 表示項目は dataset / license / source / matched_dates / item_count / asset_count / sampling_status などの透明性情報です。
  • エンドポイント上限は low が14日まで、mid が7日までです。
  • ラスタの降水量値はまだサンプリングしていません。summary.mean/max/min は実値として表示しません。
  • storm地図プレビュー、CSVのフレーム値、compareのA/B/Diff、Cardの地点カード/ミニチャート/サマリーは合成プレビューであり、観測降水量ではありません。

5. 入力例

storm 例

BBox: 139.5,35.4,140.0,35.9
期間: 2025-08-01 → 2025-08-03
プリセット: low
フレーム数: 24
対象エリア: 東京周辺
レイヤー: 地形 / 主要道路 / 避難所
メモ: Synthetic preview / not observed precipitation

compare 例

BBox: 139.5,35.4,140.0,35.9
期間A: 2025-08-01 → 2025-08-03
期間B: 2025-08-04 → 2025-08-06
プリセット: low
注: A/B/Diffは合成プレビュー、metadata到達性だけ実データ確認

card 例

緯度/経度: 35.68, 139.76
期間: 2025-08-01 → 2025-08-03
プリセット: low
metadata bbox目安: 139.51,35.43,140.01,35.93
メモ: 地点カード/ミニチャート/サマリーは合成プレビュー、metadata到達性だけ実データ確認

6. 無料枠の上限（free-ops）

項目 上限 補足
BBox 最大 5° × 5° 広い範囲は low 推奨
期間 UIプレビューは最大31日。metadata endpointは low 14日 / mid 7日 compare は A/B それぞれmetadata endpointを確認
プリセット low / mid / detail 指定外はエラーになります
storm フレーム 最大 48 超えると自動間引き

上限を超える場合は error_code: limit_exceeded が表示されます。

7. トラブルシューティング

Q. プレビューが空白になります。
A. 必須項目（モード・期間・BBox/緯度経度）が空でないか確認してください。
Q. compare のカードが表示されません。
A. compare モードに切り替え、期間A/Bをすべて入力してください。
Q. エラーコードが表示されました。
A. 無料枠の上限を超えていないか、表の上限値を確認してください。
