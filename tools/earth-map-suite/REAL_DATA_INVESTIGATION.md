# EMS-RD-01 Real Data Investigation — Earth Map Suite precipitation

Date: 2026-05-11
Scope: `tools/earth-map-suite/` plus an isolated Cloudflare Pages Functions proof endpoint at `functions/api/earth-map-suite/precipitation.js`.

> Real data is not yet enabled in the public UI unless explicitly confirmed by this investigation.

## Summary

- The public Earth Map Suite UI still uses deterministic synthetic/placeholder preview generation for `storm`, `compare`, and `card`.
- The safest candidate for a no-fixed-cost precipitation path is JAXA Earth API / EORC GSMaP daily precipitation metadata and COG/STAC assets.
- Candidate collection/dataset ID: `JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily`.
- Candidate band: `PRECIP`.
- A minimal Cloudflare Pages Function probe was added to validate request parameters and fetch only the candidate STAC collection metadata. It does **not** generate precipitation values, does **not** call paid checkout, and does **not** fall back to fake real-data values.

## Current implementation status

- `tools/earth-map-suite/app.js` is explicitly marked as placeholder logic.
- Storm output is generated by local deterministic functions (`hashString`, `mulberry32`, `generateStormFrame`, `generateStormReplay`) using the input bbox/date/preset as a seed.
- There is no browser `fetch`, XHR, JAXA SDK import, or live data request in the current public `storm` flow.
- Therefore `storm` is synthetic/placeholder preview data, not observed precipitation.

- Therefore `card` is synthetic/placeholder preview data, not observed precipitation.

## Investigated source/API path

### Official JAXA Earth API / EORC path

Official JAXA Earth API pages describe a COG/STAC-based satellite image data service for Python and JavaScript, including a precipitation category. The precipitation category is GSMaP, described as global precipitation data using the GPM core satellite and meteorological satellites, with monthly, half-monthly, and daily averages over more than 25 years since March 2000.

Official docs and sample pages identify this likely precipitation dataset path:

- Dataset/collection: `JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily`
- Band: `PRECIP`
- STAC/COG collection metadata URL pattern:
  - `https://s3.ap-northeast-1.wasabisys.com/je-pds/cog/v1/JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily/collection.json`
- Example item URL pattern seen for GSMaP monthly data:
  - `https://s3.ap-northeast-1.wasabisys.com/je-pds/cog/v1/JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_monthly/2024-04/1/E000.00-E090.00/N00.00-N90.00.json`

### Browser direct access

JAXA Earth API documentation says the JavaScript API can display satellite images in browsers and can be used to develop web apps. That means browser-direct use appears possible in principle when using the official JS API and compatible map/rendering code.

However, browser-direct integration is not the safest first production step for this repo because:

- The current Earth Map Suite is a small static tool with no external package/dependency setup.
- Real COG/STAC precipitation extraction would require the JAXA JS API or equivalent raster/COG processing, which is more than a minimal static patch.
- Browser CORS and asset-level access must be confirmed against the exact STAC/COG URLs in deployment, not assumed.
- Public UI code would need to handle upstream failures without silently showing synthetic fallback.

### Cloudflare Pages Functions requirement

Cloudflare Pages Functions are recommended for the next real-data phase and were used for the proof endpoint because they allow:

- server-side outbound fetch without browser CORS being the first blocker;
- strict input validation before any upstream request;
- a stable JSON response schema for the static UI;
- explicit `data_type: "unavailable"` error responses when upstream data is not reachable;
- no exposure of future credentials if a source later requires keys;
- no paid checkout or paid backend service.

Cloudflare Pages Functions are not necessarily required by JAXA Earth API itself, but they are the safest path for NicheWorks before public UI enablement.

## License/source requirements found

- JAXA Earth API states that API/database use is free of charge under its stated conditions and currently has no user-registration, API-key, or call-count restriction.
- JAXA Earth API states that each dataset license should be checked in the respective STAC metadata.
- JAXA Earth API states that commercial use is permitted for data to which the referenced research-data terms apply.
- JAXA/EORC disclaims accuracy/integrity/reliability/operation and asks users publishing results or software incorporating the API module to indicate API use.
- Because the exact candidate STAC license must be verified in the deployed environment, public UI real-data labels remain disabled/pending verification.

## CORS / response format / auth / package constraints

- Authentication: likely no API key for the JAXA Earth API metadata/data path per official API page, but dataset-specific terms still need STAC verification.
- Response format: JAXA Earth API uses STAC collection/item JSON and COG assets. A production endpoint must retrieve STAC items and parse COG data or delegate rendering to the official JS API.
- CORS: browser usage appears supported by the official JS API, but the exact collection/item/COG CORS behavior must be validated in Cloudflare Pages preview before UI enablement.
- Package constraints: the current repo has no root dependency setup for Functions. The proof endpoint uses only platform `fetch` and no added package. Full COG pixel extraction may require a dependency or an upstream-rendering strategy and should be handled in a separate task.
- Cost: the candidate path appears no-fixed-cost, subject to upstream availability and usage terms.

## EMS-RD-02 update — JSON-only GSMaP daily sampler

Date: 2026-05-11
Scope: `functions/api/earth-map-suite/precipitation.js` and documentation only. The public Earth Map Suite UI remains disconnected from real data.

### Verified / implemented endpoint details

- Endpoint: `/api/earth-map-suite/precipitation`
- Method: `GET` only.
- Query parameters accepted: `bbox`, `start`, `end`, `preset`. Any other query parameter is rejected with JSON.
- Dataset ID: `JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily`.
- Band: `PRECIP`.
- STAC collection endpoint used by the function:
  - `https://s3.ap-northeast-1.wasabisys.com/je-pds/cog/v1/JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily/collection.json`
- Daily STAC item URL pattern used by the function:
  - `https://s3.ap-northeast-1.wasabisys.com/je-pds/cog/v1/JAXA.EORC_GSMaP_standard.Gauge.00Z-23Z.v6_daily/YYYY-MM/DD/1/E090.00-E180.00/N00.00-N90.00.json`
  - For other bboxes, the function derives 90-degree longitude/latitude tile names such as `E000.00-E090.00/N00.00-N90.00.json`. Negative longitudes are converted to the JAXA 0..360 east-longitude tile convention.

### STAC traversal status

- The Pages Function now fetches the collection JSON first and then traverses derived daily item URLs for every requested date and intersecting 90-degree tile.
- Local syntax and mocked integration checks confirm that valid requests return `data_type: "real_observation_metadata"` when collection/item JSON contains a `PRECIP` asset.
- Live local container access to `s3.ap-northeast-1.wasabisys.com` was blocked during this task by the environment (`CONNECT tunnel failed, response 403` via `curl`, and `fetch failed` / DNS retry from Node). Therefore production/preview Cloudflare reachability must still be checked after deployment.

### PRECIP asset discovery status

- The function identifies PRECIP COG assets from STAC item `assets` by matching `PRECIP` against the asset key, title, description, roles, or href.
- If items are found but no matching asset is discovered, the endpoint returns `data_type: "unavailable"`, `status: "error"`, and `error_code: "asset_missing"`; it does not create fallback values.
- Asset discovery is verified with mocked STAC items. Live asset naming remains pending Cloudflare preview/production verification because the local container could not reach the Wasabi STAC host.

### Raster sampling status

- COG raster sampling remains **metadata-only** in RD-02.
- The endpoint returns `sampling_status: "metadata_only"` and `summary.mean/max/min: null`.
- No synthetic precipitation values are returned as real data.
- Raster sampling was not implemented because Cloudflare Pages Functions have no existing GeoTIFF/COG dependency in this repo, and adding a heavy binary or raster dependency is outside RD-02. A follow-up task should evaluate a lightweight range-read GeoTIFF parser, JAXA JS API server feasibility, or a separate no-fixed-cost sampler strategy.

### Current known limitations

- Production Cloudflare network reachability to the Wasabi-backed STAC/COG host is not yet verified by this local run.
- License text is passed through from the STAC collection when present; if absent, the endpoint reports `license_status: "needs_review"`.
- The endpoint summarizes metadata only and cannot yet calculate precipitation mean/max/min.
- Date ranges and bbox sizes are intentionally small: `preset=low` allows up to 14 days and 2.0 degrees per axis; `preset=mid` allows up to 7 days and 1.0 degree per axis.
- The public `storm`, `compare`, and `card` UI flows remain placeholder/synthetic and should not be relabeled as observed precipitation.
