AI Project Pack

AI を使った開発で、current truth、decisions、pending items、next actions、sources、update logs を一つの AI-readable repo にまとめ、毎回のセッションで文脈を作り直さなくてよくするための単一リポジトリ型運用パックです。

これは何か

AI Project Pack は、AI を使った開発のための public な単一リポジトリ型 operating pack です。ブラウザ入力ツール、メモリサービス、自律的なマネージャー、ホスト型プラットフォームではありません。主役はリポジトリそのもので、このページはそのリポジトリを先に理解するための説明ページです。

なぜ作られたか

開発プロジェクトは、決定事項がチャットに埋もれ、未決事項が勝手に確定扱いされ、source が見えなくなり、新しいモデルや新しいセッションのたびに文脈を作り直すことで劣化します。AI Project Pack は current truth、decisions、pending items、next actions、sources、append-only update logs を分離して持たせます。

リポジトリの中身

Core files は current-truth.md、decisions.md、pending.md、next-actions.md、sources.md、updates/ です。examples/、playbooks/、adapters/、docs/、templates/、validation scripts も含みます。

どう動くか

1. repo を読むREADME、AGENTS、core files を順に読んで現在地を把握します。
2. report-only を行ういきなり編集せず、まず何が整合していて何が未決かを返します。
3. bounded safe-update を1回だけ行うsafe-update は「安全を保証する更新」ではなく、「範囲を限定したレビュー可能な更新」という意味です。実行後は touched files と git diff を確認してください。
4. update log を追記する次のモデルや次のセッションがすぐ再開できるように、変更を append-only で残します。

初回の使い方

  1. リポジトリを開く
  2. README.md、AGENTS.md、core files を読む
  3. report-only を実行する
  4. bounded safe-update を1回だけ承認する
  5. touched files、git diff、根拠、権限、未決事項を確認する
  6. dated update log を追記する
  7. 次回はその repo から再開する

FAQ

AI Project Pack は Web サービスですか？

違います。主役はリポジトリそのものです。

Codex がないと使えませんか？

いいえ。リポジトリ内ファイルを読める AI なら使えます。

safe-update とは何ですか？

安全を保証する更新ではなく、範囲を限定したレビュー可能な更新という意味です。更新後は touched files と git diff を確認してください。

AIが提案した更新をそのまま反映してよいですか？

推奨しません。提案差分として扱い、diff、根拠、権限、未決事項を確認してから反映してください。

リポジトリはどこで開きますか？

リポジトリはGitHubで開きます。GitHub側の規約、プライバシーポリシー、可用性が適用されます。
