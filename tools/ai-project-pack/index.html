What it is

AI Project Pack is a public, text-first operating repo for AI-assisted development. It is not a memory service, autonomous manager, hosted platform, or browser-input utility. The product is the repository itself; this page explains how to use it before opening GitHub.

Why it exists

Projects drift when decisions stay in chat, unresolved items are treated as settled, sources disappear, and each new AI session has to reconstruct context from scratch. AI Project Pack separates current truth, decisions, pending items, next actions, sources, and append-only update logs into durable files.

What is inside the repository

Core files include current-truth.md, decisions.md, pending.md, next-actions.md, sources.md, and updates/. Guidance lives in examples/, playbooks/, adapters/, docs/, templates/, and validation scripts.

How it works

1. Read the repoStart with README, AGENTS, and the core operating files.
2. Run report-onlyRead and report before editing. This creates a bounded review checkpoint.
3. Apply one bounded safe-updateSafe-update means a bounded, reviewable update. It does not guarantee safety. Review touched files and git diff.
4. Append an update logLeave the next model a visible restart surface in updates/.

First-use flow

  1. Open the repository.
  2. Read README.md, AGENTS.md, and the core files.
  3. Run a report-only pass.
  4. Approve one bounded safe-update.
  5. Review touched files, git diff, sources, permissions, and unresolved items.
  6. Append one dated update log.
  7. Restart from the repo in the next session.

FAQ

Is AI Project Pack a website or SaaS?

No. The core product is the repository itself.

Do I need Codex to use it?

No. It can be used with AI tools that can read repository files.

What does safe-update mean?

It means a bounded, reviewable update. It does not guarantee safety. Review touched files and git diff.

Can AI-generated updates be applied directly?

No. Treat them as proposed changes and review diffs, sources, permissions, and unresolved items first.
