How to use Vibe Lexicon

Use this guide to move from rough wording to practical design language quickly. The current public version is free and optimized for fast lookup, comparison, and copy-ready wording.

1) Search

Start from the search box with rough intent words such as modern, readable, premium, or cluttered. Use suggestions to jump straight to likely terms.

2) Filter

Narrow results by category, use case, and term type when a query is too broad. This is useful when different teams use similar words for different goals.

3) Inspect detail

Select a term to open detail. Review the plain explanation, practical intent, and decomposition from vague wording to actionable language before rewriting your request.

4) Compare similar terms

Add terms to compare when wording is adjacent (for example “clean” vs “minimal,” or “premium” vs “refined”). Basic compare helps you decide which language best matches the UI outcome you want.

5) Use favorites and recent wisely

Favorites and recent history are currently local browser storage. They are convenient for repeated work on the same device and browser, but they do not sync across devices or accounts in the current public release.
