Product spec handoff

Pattern name, Japanese label, category, purpose, AI risk, user control, AI visibility, trust notes, best/not-for, required UI states, fallback behavior, and acceptance criteria.

Codex implementation task

Implementation target, expected files, UI states, accessibility, trust requirements, free behavior, Pro behavior, no AI API call, no external data send, and acceptance criteria.

GitHub Issue format

Summary, scope, implementation requirements, free/Pro behavior, UI states, accessibility requirements, QA checklist, and out of scope.

UX risk checklist

Checks whether AI looks too correct, AI generation is clear, sources and limits are visible, human review is clear, users can edit/reject/undo, failures have exits, and high-risk actions are not auto-applied.

Safety / fallback checklist

Empty, loading, partial, no useful answer, refusal, rate limit, source unavailable, manual fallback, retry, edit and retry, undo, and human handoff.

Purchase and activation

After purchase, NicheWorks Pro becomes active in this browser. It normally remains available after closing tabs or the browser. Another device, another browser, private browsing, or deleted site data requires activation again.

The tool uses the shared client /assets/nw-pro.js and NWPro.getLocalStatus() to read the Common Pro helper state. The entitlement is nicheworks_pro. localStorage is only a helper state; the server-side D1 pro_entitlements table is the source of truth.

No individual Stripe product, individual Payment Link, individual webhook, individual D1 table, or tool-specific entitlement is created for AI Interaction Atlas.
