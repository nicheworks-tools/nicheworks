Screenshot Stitcher の使い方

複数のスクリーンショットを縦につなげて、1枚の長い画像として保存するためのブラウザ完結ツールです。

1. このツールでできること

  • PNG / JPEG / WebP 画像を複数追加する
  • 画像の幅を最大幅・最小幅・指定幅に揃える
  • 上下の単色に近い余白を自動トリムする
  • PNG / WebP / JPEG で保存する
  • 長すぎる画像を指定高さごとに分割し、ZIPで保存する

2. 使い方の手順

  1. 「画像を追加」にスクリーンショットをドロップ、またはクリックして選択します。
  2. 必要に応じて「幅揃えモード」「指定幅」「画像間余白」「背景」「出力形式」を調整します。
  3. 余白を削りたい場合だけ「上下の単色余白を自動トリム」を有効にします。
  4. 「プレビュー生成」を押して、結合結果を確認します。
  5. 問題なければ「ダウンロード」または「分割ZIP出力」を押します。

3. 対応 / 非対応

対応形式は PNG / JPEG / WebP です。HEIC、BMP、TIFF、PDF には対応していません。

HEIC画像は、先に WebP / AVIF Converter などで PNG / JPEG / WebP に変換してから使ってください。

4. エラーと警告について

  • 高さが 20000px を超える場合は、分割出力を推奨します。
  • 高さが 30000px を超える場合は、分割ZIP出力を強く推奨します。
  • 総ピクセル数が大きい場合、スマホや古いブラウザでは失敗することがあります。
  • 非対応形式を選んだ場合、そのファイルはスキップされます。

5. 出力形式の説明

  • PNG：文字やUIのスクショ向け。画質劣化が少ない代わりにファイルサイズが大きくなりやすいです。
  • WebP：ファイルサイズを抑えやすい形式です。対応していない古い環境では開けない場合があります。
  • JPEG：写真向けです。文字やUIの細部は少しにじむ場合があります。

6. 分割ZIPの使いどころ

長すぎる画像は、一部SNS・ブラウザ・画像ビューアで開けないことがあります。分割高さを指定すると、指定pxごとに画像を分けてZIP保存できます。

おすすめの分割高さは 12000〜18000px です。

7. 余白トリムの注意

余白トリムは、単色に近い上下余白を削る機能です。白いヘッダー、ナビ、余白まで削られる場合があります。必ずプレビューで確認してください。

8. FAQ

画像はサーバーに送信されますか？

いいえ。画像処理はブラウザ内で行われます。

個人情報は自動で隠せますか？

いいえ。結合ツールです。必要な場合は Image Redact で隠してから共有してください。

設定を変えた後、そのまま保存できますか？

いいえ。設定変更後は古いプレビューが無効になります。再度プレビューを生成してください。

9. 免責

出力結果は必ず利用者自身で確認してください。共有前に、個人情報・アカウント名・住所・通知内容などが残っていないか確認してください。
