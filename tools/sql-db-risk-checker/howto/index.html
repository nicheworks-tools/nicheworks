SQL / DB 実行前リスクチェッカー：howto

「WHERE抜けで全件更新/削除」を実行前に止める。

広告枠（準備中）

このツールでできること

  • 検出WHERE抜け、全件UPDATE/DELETEの疑い
  • 検出TRUNCATE / DROP / ALTER などの破壊的操作
  • 表示危険度（Low/Medium/High）と理由、ハイライト
  • 安全ブラウザ完結・送信なし（SQLは保存しない）
ツールを開く 詳しい使い方（usage）

よくある症状（検索される言い方）

  • 「UPDATEにWHERE付け忘れた」
  • 「DELETEが全件削除になった」
  • 「本番でSQL事故が怖い」
  • 「DROP/TRUNCATE実行前に危険チェックしたい」
  • 「SQLの危険度を実行前に判定したい」

関連キーワード：SQL WHERE 抜け / 全件更新 防止 / 全件削除 防止 / 実行前 チェック / SQL risk checker

使い方（最短3ステップ）

  1. 実行予定のSQLを貼り付け（複数文OK）
  2. 必要なら環境/DB種別などのオプションを選択
  3. リスクチェック → 危険度と理由、ハイライトを確認

目的は「実行を止める」こと。SQLの自動修正はしない（判定と理由の提示のみ）。

注意（できないこと・限界）

  • DBの実データは見ないので、最終的な影響範囲は別途確認が必要。
  • 方言SQL/方言関数など、DB固有表現は誤判定の可能性あり。
  • 「安全」判定でも、必ず実行計画/トランザクションで最終確認推奨。
